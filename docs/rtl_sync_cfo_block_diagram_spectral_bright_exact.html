1. Board-Level Test Architecture

The ZCU102 board test wrapper exposes AXI-Lite registers and BRAM-style sample memory, then drives the internal AXI-Stream DUT. The Step 29F fix delays dut_start until the source FSM reaches SRC_STREAM.

frac_cfo_sync_bram_test_wrapper.v

2. Core DSP Pipeline

The current core is a frame-buffered functional pipeline. It estimates timing and fractional CFO from the captured frame, then applies NCO-driven complex rotation to produce a corrected output frame.

The board wrapper maps control/status registers and sample memories into an AXI-Lite accessible address space.

Board bring-up checkpoint

The first thing to confirm after rebuilding bitstream/XSA is that the board is running the updated wrapper, not a stale bitstream.

  • DEBUG_STATE[31:28] should indicate the new wrapper/debug version.
  • INPUT_COUNT > 0 proves that the source stream and dut_start are aligned.
  • If input increases but output does not, debug the DUT FSM boundary and frame detection path.
